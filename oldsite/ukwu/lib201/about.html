Coding

This course website is designed to comply with Web standards established by the W3C (World Wide Web Consortium) for HTML, XHTML, XML, CSS, XSLT, etc., and it aims to follow the best practices for accessible sites using valid and semantically-correct code. This site currently uses valid XHTML Strict 1.0 for markup and valid CSS for control of typography, color, margins, and other presentational elements. All pages and CSS files within the site are checked and validated using W3C's online (X)HTML and CSS Validators. To achieve this goal, the instructor/Web designer

  • uses the correct DOCTYPE to enable browsers to parse the pages and render them accurately
  • validates the code to correct errors or ambiguities related to syntax
  • separates style (look, appearance, colors, fonts, layouts) from content by using CSS (Cascading Style Sheets)

Accessibility

This site follows the W3C Web Content Accessibility Guidelines in an effort to remove barriers that restrict access to the Internet or its content. Such barriers include visual impairments, color blindness, mobility, learning impairments, and access barriers imposed by older technology and unconventional user agents, such as wireless devices, cellular phones, laptops, and palm devices. This site

  • uses CSS-driven relative font size for general content, which allows users to control the size of the page content
  • has alternative text for all images and decorative/positioning graphics and uses "ALT" attributes, which are important for text-based or screen-reading browsers
  • provides additional navigational aids
  • has meaningful page titles
  • identifies the content language
  • has detailed and descriptive metadata
  • whenever possible, presents main content first within the source to accommodate text-based and screen-reading browsers (please note that the navigation bar is the last division coded in a page)

In order to extend accessibility to multiple media, all pages are linked to the following external style sheets:

  • aural style sheet coded for speech-synthesizing devices
  • Braille style sheet coded for braille-tactile devices
  • embossed style sheet coded for braille-printing devices
  • handheld style sheet coded for handheld devices
  • TTY style sheet coded for fixed-pitch teletypes

The pages are run across Web-based accessibility checkers, such as Bobby and Cynthia Says Portal to achieve good coding practices.
